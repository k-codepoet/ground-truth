# 외부 표준 기반 플러그인 동기화 전략

## 핵심 원칙

외부 표준/규격에 의존하는 플러그인은 **주기적 동기화**가 필수다. 누락된 사양은 정상 동작을 보장하지 못한다.

## 문제 상황

- 표준이 활발하게 개선 중일 때 (Claude Code Plugins 등)
- 플러그인이 해당 표준의 가이드 역할을 할 때
- 공식 문서 변경이 플러그인 품질에 직접 영향을 미칠 때

## 해결 패턴

### 1. 비교 리서치 수행
공식 문서와 현재 플러그인 내용을 대조하여 차이점 파악

### 2. 차이 유형 분류
| 유형 | 조치 |
|------|------|
| 누락된 필드/이벤트 | 추가 |
| 공식에 없는 내용 | 제거 또는 검증 필요 표시 |
| 설명 불일치 | 공식 기준으로 수정 |

### 3. 검증 프로세스
1. 웹검색으로 공식 문서 확인
2. 플러그인 파일 수정
3. `/ced:validate`로 검증

## 적용 사례

- **ced 플러그인 (2026-01-02)**: Command Guide에 `model`, `disable-model-invocation` 추가, Hook Guide에 4개 이벤트 추가 및 미지원 타입 제거

## 권장 주기

표준이 활발히 개선 중이면 **버전 업데이트 시마다** 동기화 검토
