## Principle
Go/No-Go 체크

## Why
미래 대비용 작업에 시간을 낭비하지 않고, 정말 중요한 일에 집중하기 위해.
YAGNI (You Aren't Gonna Need It) 원칙의 실천적 적용.

**핵심**: 기록은 자유롭게, 실행은 신중하게

## How

### 언제 적용하나?

- 금방 될 일 (5분 내) → 체크 불필요, 그냥 진행
- 깊어질 것 같은 작업 → 체크 필요
  - 조사/설계가 필요한 작업
  - 여러 파일을 건드리는 작업
  - "체계를 잡아두자" 류의 작업

### 체크리스트

시간이 많이 들 것 같은 작업 시작 전:

```
□ 실제 문제가 발생했는가? (미래 걱정 아닌지)
□ 현재 사용자가 나만인가?
□ 문제가 발생하면 그때 해도 늦지 않은가?
```

### 판정

- 3개 모두 "아니오" → **Go** (진행)
- 하나라도 "예" → **No-Go** (즉시 중단, 우선순위 재점검)

## Examples

### No-Go 사례
플러그인 간 버전/종속성 관리 체계 조사:
- 실제 문제가 발생했는가? → 아니오 (미래 걱정)
- 현재 사용자가 나만인가? → 예
- 문제가 발생하면 그때 해도 늦지 않은가? → 예

→ **No-Go**: 즉시 중단. 나중에 실제 문제가 생기면 그때 처리.

### Go 사례
버그 수정, 기능 추가 등 실제 필요에 의한 작업:
- 실제 문제가 발생했는가? → 예
- 현재 사용자가 나만인가? → 해당 없음
- 문제가 발생하면 그때 해도 늦지 않은가? → 아니오

→ **Go**: 진행.
