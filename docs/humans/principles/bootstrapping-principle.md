## Principle
계단식 부트스트래핑 원칙

## Why
선언형으로 더 높은 일을 하려면, 도구들이 계단식으로 쌓여야 한다. AI company를 만들기 위해서는 도구가 스스로를, 그리고 서로를 발전시킬 수 있어야 한다.

## How

```
[메타 원칙] 계단식 진행
         │
    ┌────┴────┐
[구축]        [운영]
독자적 가치    위임 가능성
상호 발전      신뢰 기반 위임
```

### 메타 원칙: 계단식 진행
바로 안 되면 한 단계씩 쌓는다. 모든 원칙을 관통.

### 구축 원칙 (무엇을 만들 때)
- **독자적 가치**: 각 단계가 그 자체로 쓸모 있다
- **상호 발전**: 도구들이 서로를 고친다

### 운영 원칙 (어떻게 넘길 때)
- **위임 가능성**: 수동 → HITL → 기준 → Agent
- **신뢰 기반 위임**: 직접 해본 후에만 넘긴다
  - 3번 이상 직접 해봤는가?
  - 패턴을 기준으로 세울 수 있는가?
  - 입력 → 결과가 예상 가능한가?
  - (셋 다 만족해야 위임 가능)

## Examples

### gemify ↔ forgeify 순환

```
gemify (지식 생산)          forgeify (코드 실행)
    │                            │
    ├─ improve-plugin 문서 ──────→ 코드 수정
    │                            │
    ←── 플러그인 검수/버그픽스 ────┘
```

- gemify로 조사, 계획, 의사결정
- forgeify로 코드 수정, 검수, 버그픽스
- 둘이 서로를 점진적으로 개선
- 사람(HITL)이 방향성과 중심을 잡음

### Stage 진화

```
Stage 0 (수동) → Stage 1 (AI 보조/HITL) → Stage 2 (AI 주도) → Stage 3 (AI 자율)
```

현재 Stage 1을 깊게 수행하며 Stage 2 진입 재료를 수집 중.
