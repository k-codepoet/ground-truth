# 지식 파이프라인 단계 명명 혼란 개선

> 날짜: 2025-12-31
> 출처: 대화

---

## 문제 인식

seed → growing → digest → corpus 3단계 파이프라인에서:

1. **명명 불일치**: `/seed` 명령이 `inbox/`로 저장됨 → `seed/` 폴더가 있어야 할 것 같은 기대와 불일치
2. **진행 상태 추적 어려움**: 단계별로 처리된다는 것을 사람들이 확인할 방도 필요

## 고민되는 방향

### 옵션 1: 번호 붙이기
- `1-seed/`, `2-growing/`, `3-digest/` 같은 prefix
- 순서가 명확해짐

### 옵션 2: 폴더 안에 넣어서 관리
- `pipeline/seed/`, `pipeline/growing/` 같은 구조
- 상위 폴더로 묶어서 관리

### 현재 구조 vs 기대
```
현재: inbox/ → growing/ → corpus/
기대: seed/ → growing/ → corpus/ (명령어와 일치)
```

## 핵심 질문

- 명령어 이름(`/seed`)과 폴더 이름(`inbox`)이 일치해야 직관적인가?
- 아니면 역할 중심(`inbox`=받은편지함)이 더 명확한가?
