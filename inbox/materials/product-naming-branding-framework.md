# 제품 네이밍 & 브랜딩: 스토리와 구조의 분리 원칙

> **Type**: Material (외부 재료)
> **Source**: Gemify 브랜딩 실제 의사결정 과정에서 도출
> **Purpose**: 제품 네이밍/브랜딩 시 참고할 프레임워크

---

## 1. 핵심 원칙

### 분리의 법칙

```
┌─────────────────────────────────────────────────────────────┐
│  브랜드 레이어 (감성, 스토리)                                 │
│  "Turn your thoughts into gems"                             │
│  → 마케팅, 랜딩페이지, 첫인상                                 │
└─────────────────────────────────────────────────────────────┘
                         ↑ 분리 ↑
┌─────────────────────────────────────────────────────────────┐
│  UX 레이어 (직관, 기능)                                      │
│  inbox/ → drafts/ → library/                                │
│  → 실제 사용, 학습 곡선 제로                                  │
└─────────────────────────────────────────────────────────────┘
```

**왜 분리해야 하는가?**

| 문제 상황 | 결과 |
|----------|------|
| 브랜드 메타포를 UX에 강제 | "ore? flux? 이게 뭐야?" → 이탈 |
| UX가 너무 평범 | 기억 안 됨, 차별화 실패 |
| 둘 다 강한 메타포 | 학습 곡선 ↑, 설명 비용 ↑ |

**정답**: 브랜드는 기억에 남게, UX는 생각 없이 쓸 수 있게.

## 2. 브랜드 레이어 설계

### 2.1 좋은 브랜드 메타포의 조건

| 조건 | 설명 | 체크 |
|------|------|:----:|
| **보편성** | 타겟 전체가 이해 | ☐ |
| **일관성** | 전체 스토리가 하나의 이미지 | ☐ |
| **의미 정확성** | 제품이 하는 일과 일치 | ☐ |
| **문화적 적합성** | 타겟 문화권에서 자연스러움 | ☐ |
| **확장성** | 기능 추가 시 메타포 안에서 설명 가능 | ☐ |

### 2.2 메타포 선택 프로세스

```
Step 1: 제품의 핵심 가치 한 문장 정의
        "잡생각을 정제해서 가치 있는 지식으로"
                    ↓
Step 2: 핵심 동작 추출
        "정제", "가치 상승", "변환"
                    ↓
Step 3: 후보 메타포 브레인스토밍
        - 식물: seed → grow → harvest
        - 술: gather → ferment → distill
        - 광물: rough → refine → gem
        - 요리: gather → mix → bake
                    ↓
Step 4: 타겟별 필터링
        - 연령 제한? (술 메타포 탈락)
        - 성별 편향? (대장장이 메타포 약점)
        - 문화 장벽? (corpus 같은 전문 용어)
                    ↓
Step 5: 의미 정확성 검증
        - "harvest"는 단순 수확 ≠ 정제
        - "distill"은 정제 ✓ but 술 연상
        - "gem"은 가치 상승 + 정제 ✓
                    ↓
Step 6: 최종 선택 + 태그라인
```

### 2.3 메타포 강도 스펙트럼

```
강한 메타포                              직관적
(독특, 학습 필요)                    (평범, 즉시 이해)
     │                                    │
     ▼                                    ▼
ore/flux/refining ◄─────────────► inbox/drafts/library
     │                                    │
     └── 브랜딩에만 사용 ──────────────────┘ UX에 사용
```

## 3. UX 레이어 설계

### 3.1 좋은 UX 용어의 조건

| 조건 | 설명 | 체크 |
|------|------|:----:|
| **즉시 이해** | 설명 없이 의미 파악 | ☐ |
| **기존 경험 활용** | 이메일의 inbox, 워드의 drafts | ☐ |
| **시맨틱 명확성** | AI/기계도 의미 파악 가능 | ☐ |
| **국제화 용이** | 번역해도 의미 유지 | ☐ |

### 3.2 성공 제품 패턴 분석

| 제품 | 브랜드 | UX 용어 | 전략 |
|------|--------|---------|------|
| Notion | "All-in-one workspace" | Page, Database, Block | 완전 직관적 |
| Obsidian | "A second brain" | Vault, Note, Link | Vault만 브랜드, 나머지 직관 |
| Linear | "The issue tracking tool" | Issue, Project, Cycle | 직관적 |
| Figma | "Design together" | File, Frame, Component | 직관적 |
| Git | (기술 도구) | commit, branch, merge | 학습 필요하지만 표준화됨 |

**패턴**: 대부분 UX는 직관적, 브랜드는 태그라인/마케팅에만.

### 3.3 타겟별 용어 검증 매트릭스

용어 선택 시 모든 타겟 그룹에서 테스트:

```
| 용어 | 일반인 | 크리에이터 | 개발자 | 연구자 | AI |
|------|:------:|:----------:|:------:|:------:|:--:|
| [후보1] | ? | ? | ? | ? | ? |
| [후보2] | ? | ? | ? | ? | ? |

✅ = 즉시 이해
⚠️ = 약간의 설명 필요  
❌ = 혼란 유발
```

## 4. 네이밍 패턴

### 4.1 제품 이름 유형

| 유형 | 예시 | 장점 | 단점 |
|------|------|------|------|
| 단일 단어 | Notion, Slack, Figma | 강렬, 기억 쉬움 | 상표 확보 어려움 |
| 조합어 | YouTube, Facebook | 의미 전달 + 독특 | 길어질 수 있음 |
| 동사화 (-ify) | Shopify, Spotify, **Gemify** | 행동 암시, 변환 느낌 | 트렌디해 보임 |
| 형용사+명사 | QuickBooks, Evernote | 특성 강조 | 평범할 수 있음 |

### 4.2 "-ify" 패턴의 힘

```
[명사] + ify = "[명사]로 만들다"

Shopify = Shop으로 만들다 (쇼핑몰 구축)
Spotify = Spot(발견) + ify (음악 발견)
Gemify = Gem으로 만들다 (생각을 보석으로)
```

**장점**:
- 동사라서 행동 유도
- 변환/생성 의미 내포
- 검색 구별 용이
- 현대적 느낌

### 4.3 태그라인 공식

| 공식 | 예시 |
|------|------|
| [동사] your [대상] into [결과] | "Turn your thoughts into gems" |
| [대상] deserves to [동사] | "Your thoughts deserve to shine" |
| From [before] to [after] | "From rough ideas to polished knowledge" |
| [동사]. [동사]. [동사]. | "Capture. Develop. Shine." |

## 5. 문화적 검증 체크리스트

### 5.1 미국/영어권 기준

| 체크 항목 | 질문 |
|----------|------|
| 발음 | 쉽게 발음되는가? 어색한 소리 조합 없는가? |
| 기존 의미 | 부정적 의미, 슬랭, 비속어 아닌가? |
| 상표 충돌 | 기존 유명 제품/서비스와 혼동되지 않는가? |
| 관용 표현 | 기존 표현과 자연스럽게 연결되는가? |
| 검색 가능성 | 검색 시 원하는 결과가 나오는가? |

### 5.2 흔한 영어 표현 활용

브랜드 메타포가 기존 관용 표현과 연결되면 강력:

| 표현 | 의미 | 활용 |
|------|------|------|
| "Diamond in the rough" | 숨겨진 가치 | 원석 → 보석 스토리 |
| "Polish your skills" | 기술 연마 | 정제 과정 |
| "Hidden gem" | 숨겨진 보석 | 발견의 가치 |
| "Plant a seed" | 시작하다 | 식물 메타포 |
| "Distill your thoughts" | 생각 정리 | 증류 메타포 |

## 6. 실전 적용 프레임워크

### Phase 1: 핵심 정의 (30분)

```markdown
1. 제품이 하는 일 (한 문장):
   ___________________________________

2. 핵심 동작/가치 (3개):
   - ___________
   - ___________
   - ___________

3. 타겟 그룹:
   □ 일반인  □ 크리에이터  □ 개발자  □ 연구자  □ AI
```

### Phase 2: 메타포 탐색 (1시간)

```markdown
후보 메타포 (최소 3개):

| 메타포 | 흐름 | 핵심 가치 연결 |
|--------|------|----------------|
| 1.     |      |                |
| 2.     |      |                |
| 3.     |      |                |

각 후보별 필터링:
- 연령 제한? Y/N
- 성별 편향? Y/N
- 문화 장벽? Y/N
- 의미 정확? Y/N
```

### Phase 3: UX 용어 결정 (30분)

```markdown
핵심 폴더/기능 명칭:

| 기능 | 후보 A | 후보 B | 선택 |
|------|--------|--------|------|
| 입력 |        |        |      |
| 처리 |        |        |      |
| 저장 |        |        |      |

타겟별 검증 완료? □
```

### Phase 4: 네이밍 (1시간)

```markdown
이름 후보 (최소 5개):
1. ___________
2. ___________
3. ___________
4. ___________
5. ___________

검증:
- 발음 테스트 □
- 상표 검색 □
- 도메인 확인 □
- 기존 의미 확인 □
```

### Phase 5: 태그라인 (30분)

```markdown
태그라인 후보 (최소 3개):
1. ___________
2. ___________
3. ___________

브랜드 메타포와 연결? □
행동 유도? □
기억하기 쉬움? □
```

## 7. 안티패턴 (피해야 할 것)

| 안티패턴 | 문제 | 예시 |
|----------|------|------|
| 메타포 혼재 | 이미지 혼란 | Capture(사냥) + Cultivate(농업) + Corpus(학술) |

| 문화적 맹점 | 타겟 이탈 | 술 메타포 → 미성년자 제외 |

## 8. Gemify 사례 요약

### Before (문제)
- Seed → Grow → Digest (메타포 깨짐)
- ore/, flux/, corpus/ (UX에 강한 메타포)
- "Gem" (검색 구별 어려움)

### After (해결)
- 브랜드: 💎 광물/보석 메타포 ("rough to gem")
- UX: inbox/, drafts/, library/ (직관적)
- 이름: Gemify ("-ify" 변환 패턴)
- 태그라인: "Turn your thoughts into gems"

### 핵심 학습
> **브랜드는 기억에 남게, UX는 생각 없이 쓸 수 있게.**
