# 6대 Domain이 지식 관리체계와 맞지 않는 문제

## 맥락

ground-truth 프로토콜 정의 중 CLAUDE.md progressive disclosure 적용하면서 발견.

## 현재 6대 Domain

| Domain | 핵심 질문 |
|--------|----------|
| product | 무엇을 만들 것인가? |
| engineering | 어떻게 만들 것인가? |
| operations | 어떻게 돌릴 것인가? |
| growth | 어떻게 알릴 것인가? |
| business | 어떻게 유지할 것인가? |
| ai-automation | 어떻게 위임할 것인가? |

## 문제

- library 분류 기준으로 6대 domain 사용 중
- 그런데 실제 지식 관리 흐름과 맞지 않는 느낌
- "6대 domain은 views에서 필요로 하는 핵심 기준"이 아닐까?

## 관련 논의

- library = 재료 (재사용 가능한 개념)
- views = 레시피 + 결과물 연결
- domain은 views의 "왜/뭘/어디까지"를 구조화하는 기준?
