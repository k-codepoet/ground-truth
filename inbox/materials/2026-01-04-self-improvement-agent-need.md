# 자가 발전 에이전트 필요성

## 맥락

library 분류 체계 리팩토링 작업 중, 관련 문서들(draft, 개선 문서, CLAUDE.md 등)을 일일이 지시해서 업데이트해야 했음.

## 관찰된 패턴

1. draft에서 결정 확정
2. 관련 개선 문서 업데이트 필요 → 사용자가 지시
3. gemify plugin 개선 문서에도 반영 필요 → 사용자가 지시
4. workflows/ 타입 추가 → 사용자가 지시
5. ...

**문제**: 사용자가 일일이 "이것도 해야지"라고 지시해야 함. 놓치면 불일치 발생.

## 사용자 요구

> "이런 걸 발견해서 나에게 역제안하고 자동으로 해주게 gemify에 subagent를 만들고 싶어.
> 자동으로 자가 발전해야지. 내가 놓쳐도. 제안은 하되 내가 컨펌하는 식으로."

## 핵심 요구사항

- **자동 발견**: 변경 시 영향받는 문서/설정 자동 탐지
- **역제안**: Claude가 먼저 "이것도 업데이트해야 할 것 같아요" 제안
- **컨펌 기반**: 사용자 승인 후 실행 (HITL 유지)
- **자가 발전**: 시스템이 스스로 개선점 찾아냄
