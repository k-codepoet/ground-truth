# Inbox 탐색 및 우선순위 분석 세션

## 발단

새 세션에서 inbox에 처리 안 된 내용이 있는지 탐색하고 우선순위에 맞게 작업 준비하는 요청.

## 수행한 분석

### 1단계: 파일 수집
- `inbox/**/*.md` glob으로 전체 파일 목록 수집
- `status: raw` grep으로 미처리 파일 필터링
- `status: used` grep으로 처리 완료 파일 확인

### 2단계: 내용 분석
- 각 raw 파일의 frontmatter와 본문 읽기
- `references` 필드로 파일 간 연결 관계 파악

### 3단계: 클러스터링
- 연관된 주제끼리 그룹화
- 예: plugin-validator + plugin-sync + ced-improve → "플러그인 개선 클러스터"

### 4단계: 우선순위 판단
- 연결성 높은 클러스터 → 1순위
- 단독 발전 가능 → 2순위
- 별도 주제/참조용 → 3순위

## 산출물 형태

| 섹션 | 내용 |
|------|------|
| 현황 요약 | raw 파일 개수, 진행 중 drafts 여부 |
| 클러스터별 분류 | 연관 파일 그룹 + 추천 액션 |
| 우선순위 테이블 | 파일, 내용, 도메인/비고 |

## 사용자 피드백

- "이미 처리된 것들도 나열된 거 같은데?" → `status: used` 체크 누락 지적
- 수정 후 "클러스터링해서 알려준 것도 좋네" → 클러스터링 방식 긍정 평가
- "새로운 사람이 작업 인계받을 때 쓸 수 있게 스킬/커맨드로 만들고 싶다"

## 핵심 요구사항

1. inbox의 raw 파일만 필터링
2. 파일 간 references 기반 연결 관계 분석
3. 주제별 클러스터링
4. 우선순위 제안
5. 추천 다음 액션 제시
