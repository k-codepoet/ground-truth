## 아이디어

세션 종료 시점에 호출하는 전용 스킬/커맨드 필요.

## 기능 구체화

### `/gemify:wrapup` 또는 `/gemify:close`

1. **놓친 것 체크**
   - 이번 세션에서 다룬 파일들 리스트
   - 완료/미완료 상태 확인
   - 열린 질문(Open Questions) 정리

2. **종결 리포트 생성**
   - 세션 요약 (무엇을 했는가)
   - 결과물 목록 (생성/수정된 파일)
   - inbox에 저장해둔 것들 목록 (다음 작업용)

3. **다음 액션 제안**
   - 이번 세션에서 파생된 TODO
   - inbox에 저장한 것 중 우선순위 높은 것
   - "다음에 뭐 해야 하지?" 질문에 바로 답변 가능하게

4. **HITL 관점 활용**
   - 리포트를 보고 사람이 최종 검토
   - 누락된 것 있으면 추가 지시
   - "이 정도면 됐다" 승인 후 종료

## 왜 필요한가

- 부트스트래핑 원칙의 "신뢰 기반 위임" 적용
- 반복되는 패턴 → 규칙화 → 스킬로 위임
- 세션마다 일관된 품질의 마무리 보장

## 중요: inbox는 "남은 것"이 아님

- inbox에 저장된 것들은 이 대화 맥락에 "남은 것"이 아님
- 다음 작업을 위해 **의도적으로 챙겨둔 것**
- wrapup 스킬에서 이 구분을 명확히 해야 함
  - "미완료 작업" vs "다음을 위해 저장한 것"
  - inbox 항목은 후자로 취급
