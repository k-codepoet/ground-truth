## 핵심 아이디어

대화 맥락을 모니터링하다가 "기록할 만한 순간"을 감지하면 자동으로 capture-pair를 제안하는 에이전트.

## 감지해야 할 패턴

### 높은 우선순위
- **오류 → 해결** 패턴: 에러 발생 후 수정 완료
- **의사결정 완료**: "이걸로 하자", "A로 결정"
- **반복 문제 언급**: "매번", "또", "항상 이런다"
- **개선 필요 언급**: "나중에 고쳐야", "TODO"

### 중간 우선순위
- **새로운 발견**: "알았다", "이거였네", "원인 찾았다"
- **학습 포인트**: "몰랐는데", "처음 알았다"

## 에이전트 동작

```yaml
name: insight-watcher
description: 대화에서 기록할 만한 인사이트를 감지하고 capture-pair 제안

triggers:
  - 오류 해결 완료
  - 의사결정 확정
  - 반복 패턴 발견
  - 개선 아이디어 도출

action:
  - 조용히 모니터링 (방해 X)
  - 적절한 순간에 제안: "이 내용을 기록해둘까요?"
  - 사용자 승인 시 capture-pair 실행
```

## 구현 고려사항

- 너무 자주 제안하면 귀찮음 → 임계값 설정
- 제안 타이밍이 중요 → 대화 흐름 끊지 않기
- SessionStart hook과 연계 가능?
